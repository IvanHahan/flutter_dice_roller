🎯 Що таке Dart 3.x

Dart — це сучасна об'єктно-орієнтована мова, яка компілюється до нативного коду (ARM/x86) або до JavaScript/WebAssembly.

Характеристики

  • Строга типізація (compile-time checked)
  • Sound null safety (Dart 2.12+)
  • AOT → нативний код; JIT → hot reload
  • Всі значення — об'єкти (успадковують Object)

Нове в Dart 3.x

  • Records — анонімні кортежі
  • Patterns — деструктуризація та зіставлення
  • Sealed classes — контроль ієрархії типів
  • Switch expressions — switch як вираз
  • Class modifiers — base, interface, mixin class
void main() {
  var point = (3, 'y'); // Record (Dart 3+)
  switch (point.$1) {
    > 0 => print('Positive'),
    _   => print('Non-positive'),
  }
}

⚠️ Коли Flutter не найкращий вибір

Ігри

Відсутність продвинутого звукового рушія. Для аудіо DAW обирайте нативну розробку.

watchOS / tvOS

🏗️ Архітектура Flutter

Framework (Dart) Material, Cupertino, Widgets, Rendering
⬇
Engine (C++) Impeller, Skia, Text, Dart VM
⬇
Embedder (Platform-specific) Android, iOS, macOS, Windows, Linux, Web

Рівні

  • Framework — бібліотеки Dart високого рівня (віджети, анімація, жести)
  • Engine — базові бібліотеки C++ (рендерінг, текст, введення/виведення)
  • Embedder — платформо-специфічний код для кожної ОС
⚡ Рушій Impeller забезпечує швидший та стабільніший рендерінг, ніж Skia, особливо на Android. На iOS Skia повністю видалено з Flutter 3.29.

▶️ Як запускається Flutter застосунок

import 'package:flutter/material.dart';

void main() {
  runApp(
    const MaterialApp(
      home: Scaffold(
        body: Center(
          child: Text('Hello World'),
        ),
      ),
    ),
  );
}
const — створює віджет на етапі компіляції, зменшуючи використання пам'яті. Додавайте const скрізь, де можливо.

Порядок виконання

  • main() — точка входу, виконується автоматично
  • runApp() — приймає кореневий віджет та рендерить його
  • MaterialApp — обгортка Material Design (тема, навігація)
  • Scaffold — базова структура екрану (AppBar, body, drawer)

🌳 Дерево віджетів

MaterialApp
  └── Scaffold
        └── Center
                └── Text("Hello World")
ℹ️ Material 3 — стандарт з Flutter 3.16. useMaterial3: true більше не потрібен.

🧱 Віджети (Widgets)

Віджет — це будівельний блок інтерфейсу користувача. Віджети вкладаються один в одного, утворюючи дерево віджетів.

  • Все у Flutter — віджет
  • Віджети описують конфігурацію, а не сам елемент на екрані
  • Flutter сам оптимізує рендерінг — ви лише описуєте що показати
🎯 Аналогія: віджети — як деталі Lego. З простих блоків збираються складні інтерфейси.

Статичні та динамічні віджети

  • StatelessWidget — незмінний. Не може оновлюватися після створення.
  • StatefulWidget — може змінювати стан через setState().
class MyWidget extends StatelessWidget {
  const MyWidget({super.key});

  @override
  Widget build(BuildContext context) {
    return const Text('Hello!');
  }
}

🌳🌳🌳 3 дерева Flutter

Flutter підтримує три паралельні дерева для оптимізації рендерінгу:

⚙️

Widget Tree

Конфігурація
Зберігає опис UI. Легке, може перебудовуватися часто. Те, з чим ви працюєте безпосередньо.

🔄

Element Tree

Життєвий цикл
Керує зв'язком між Widget та RenderObject. Відповідає за mount/unmount, ключі.

🎨

Render Tree

Рендерінг
Відповідає за розмір, позицію та малювання. Важке дерево — змінюється рідко.

Розуміння трьох дерев не обов'язкове для початку, але допомагає зрозуміти, чому Flutter швидкий: часті зміни у Widget Tree не викликають перерендерінг Render Tree.

📊 Типи даних Dart

Dart — строго типізована мова. Кожне значення — об'єкт, що успадковує Object.

Ієрархія типів

Object? (верхній тип, може бути null)
 └── Object (базовий клас для всього)
      ├── num
      │   ├── int
      │   └── double
      ├── String
      ├── bool
      └── Widget (Flutter)

Never (нижній тип — ніколи не має значення)

Приклади

15 → int → num → Object
'Hello' → String → Object
MaterialApp → Widget → Object
null → Null → Object?
🔧 Dart 3.2+: приватні final поля можуть бути типізовано підвищені (private field promotion).
В Dart 3+ верхній тип — Object? (а не Object), а нижній — Never. Це відрізняється від багатьох інших мов.

📝 Змінні в Dart

Ключове слово Змінна? Коли визначається
var / тип ✅ Так Runtime
final ❔ Одне призначення Runtime
const ❌ Ні Compile time
late ❔ Одне призначення Runtime (відкладено)
💡 Правило: const > final > var. Використовуйте найбільш обмежений варіант.
var name = 'Bob';            // автоматична типізація → String
String name = 'Bob';        // явна типізація
String? nullableName;        // може бути null (null safety)
final name = 'Bob';          // runtime-константа
const bar = 1000000;         // compile-time константа
late final String label;      // ініціалізація при першому доступі

// Dart 3.2+: private field promotion
class Example {
  final String _name;  // може бути promoted як public
  
  @override
  String toString() => _name.toUpperCase();
}
